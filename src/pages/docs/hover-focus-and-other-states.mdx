Cada clase de utilidad en Tailwind se puede aplicar _condicionalmente_ agregando un modificador al comienzo del nombre de la clase que describe la condición a la que deseas apuntar.

Por ejemplo, para aplicar la clase `bg-sky-700` al pasar el mouse, usa la clase `hover:bg-sky-700`:

<Example hint="Hover over this button to see the background color change">
  <div class="grid place-items-center">
    <button class="bg-sky-500 hover:bg-sky-700 px-5 py-2 text-sm leading-5 rounded-full font-semibold text-white">
      Guardar cambios
    </button>
  </div>
</Example>

```html
<button class="bg-sky-600 **hover:bg-sky-700** ...">
  Guardar cambios
</button>
```

<details className="-mt-2 mb-10 rounded-xl border px-6 py-3 prose prose-slate open:pb-5 dark:prose-dark dark:border-slate-800">
  <summary className="font-medium cursor-default select-none text-slate-900 dark:text-slate-200">¿Cómo se compara esto con el CSS tradicional?</summary>

Al escribir CSS de la manera tradicional, un solo nombre de clase haría cosas diferentes según el estado actual.

<TipBad>Tradicionalmente, el mismo nombre de clase aplica diferentes estilos al pasar el mouse</TipBad>

```css
.btn-primary {
  background-color: #0ea5e9;
}
.btn-primary:hover {
  background-color: #0369a1;
}
```

En Tailwind, en lugar de agregar los estilos para un estado hover a una clase existente, agregas otra clase al elemento que _solo_ hace algo al pasar el mouse.

<TipGood>En Tailwind, se usan clases separadas para el estado predeterminado y el estado hover</TipGood>

```css
.bg-sky-500 {
  background-color: #0ea5e9;
}
.hover\:bg-sky-700:hover {
  background-color: #0369a1;
}
```

¿Ves cómo `hover:bg-sky-700` _solo_ define estilos para el estado `:hover`? No hace nada de forma predeterminada, pero tan pronto como pasas el cursor sobre un elemento con esa clase, el color de fondo cambiará a `sky-700`.

Esto es lo que queremos decir cuando decimos que una clase de utilidad se puede aplicar _condicionalmente_: al usar modificadores, puedes controlar exactamente cómo se comporta tu diseño en diferentes estados, sin tener que abandonar tu HTML.

</details>

Tailwind incluye modificadores para casi todo lo que necesitarás, incluyendo:

- [Pseudo-clases](#pseudo-clases), como `:hover`, `:focus`, `:first-child` y `:required`
- [Pseudo-elementos](#pseudo-elementos), como `::before`, `::after`, `::placeholder`, y `::selection`
- [Media queries](#media-queries), como breakpoints responsive, modo oscuro y `prefers-reduced-motion`
- [Selectores de atributos](#selector-de-atributos), como `[dir="rtl"]` y `[open]`

Estos modificadores pueden incluso [apilarse](#ordenar-modificadores-apilados) para dirigirse a situaciones más específicas, por ejemplo, cambiar el color de fondo en modo oscuro, en el breakpoint medio, al pasar el mouse:

```html
<button class="**dark:md:hover:bg-fuchsia-600** ...">
  Guardar cambios
</button>
```

En esta guía, aprenderás sobre todos los modificadores disponibles en el framework, cómo usarlos con tus propias clases personalizadas e incluso cómo crear las tuyas propias.

## Pseudo-clases

### Hover, focus, y active

Estila elementos al pasar el mouse, enfocar y activar usando los modificadores `hover`, `focus` y `active`:

<Example hint="Try interacting with this button to see the hover, focus, and active states">
  <div class="grid place-items-center">
    <button type="button" class="bg-violet-500 hover:bg-violet-600 focus:outline-none focus:ring focus:ring-violet-300 active:bg-violet-700 px-5 py-2 text-sm leading-5 rounded-full font-semibold text-white">
      Guardar cambios
    </button>
  </div>
</Example>

```html
<button class="bg-violet-500 **hover:bg-violet-600** **active:bg-violet-700** **focus:outline-none focus:ring focus:ring-violet-300** ...">
  Guardar cambios
</button>
```

Tailwind también incluye modificadores para otros estados interactivos como `:visited`, `:focus-within`, `:focus-visible` y más.

Consulta la [referencia de pseudoclase](#referencia-de-pseudoclase) para obtener una lista completa de los modificadores de pseudoclase disponibles.

### Estilo basado en el estado de hermano <small>(peer-{'{modifier}'})</small>

Cuando necesites diseñar un elemento en función del estado de un elemento _hermano_, marca el hermano con la clase `peer` y usa modificadores `peer-*` como `peer-invalid` para diseñar el elemento de destino:

<Example hint="Try making the email address valid to see the warning disappear" p="none">
  <div class="max-w-sm mx-auto bg-white shadow pt-6 pb-5 px-6">
    <form>
      <div>
        <label for="email" class="block text-sm font-medium text-slate-700">Email</label>
        <div class="mt-1">
          <input type="email" name="email" id="email" class="peer px-3 py-2 bg-white border shadow-sm border-slate-300 placeholder-slate-400  disabled:bg-slate-50 disabled:text-slate-500 disabled:border-slate-200 focus:outline-none focus:border-sky-500 focus:ring-sky-500 block w-full rounded-md sm:text-sm focus:ring-1 invalid:border-pink-500 invalid:text-pink-600 focus:invalid:border-pink-500 focus:invalid:ring-pink-500 disabled:shadow-none" value="george@krugerindustrial." placeholder="you@example.com" />
          <p class="mt-2 invisible peer-invalid:visible text-pink-600 text-sm">Por favor ingresa una dirección de correo electrónico válida.</p>
        </div>
      </div>
    </form>
  </div>
</Example>

```html
<form>
  <label class="block">
    <span class="block text-sm font-medium text-slate-700">Email</span>
    <input type="email" class="**peer** ..."/>
    <p class="mt-2 invisible **peer-invalid:visible** text-pink-600 text-sm">
      Por favor ingresa una dirección de correo electrónico válida.
    </p>
  </label>
</form>
```

Esto hace posible hacer todo tipo de trucos ingeniosos, como [etiquetas flotantes](https://www.youtube.com/watch?v=nJzKi6oIvBA), por ejemplo, sin JS.

Este patrón funciona con todos los modificadores de pseudoclase, por ejemplo, `peer-focus`, `peer-required` y `peer-disabled`.

Es importante tener en cuenta que el marcador `peer` solo se puede usar en hermanos _anteriores_ debido a cómo funciona el [combinador general de hermanos](https://developer.mozilla.org/en-US/docs/Web/CSS/General_sibling_combinator) en CSS.

<TipBad>No funcionará, solo los hermanos anteriores pueden marcarse como compañeros</TipBad>

```html
<label>
  <span class="**peer-invalid:text-red-500** ...">Email</span>
  <input type="email" class="**peer** ..."/>
</label>
```

## Pseudo-elementos

### Before y after

Dale estilo a los pseudo-elementos `::before` y `::after` usando los modificadores `before` y `after`:

<Example p="none">
  <div class="max-w-sm mx-auto bg-white shadow pt-6 pb-5 px-6">
    <form>
      <label class="block">
        <span class="block text-sm font-medium text-slate-700 after:content-['*'] after:ml-0.5 after:text-red-500">Email</span>
        <input type="email" name="email" id="email" class="mt-1 px-3 py-2 bg-white border shadow-sm border-slate-300 placeholder-slate-400 focus:outline-none focus:border-sky-500 focus:ring-sky-500 block w-full rounded-md sm:text-sm focus:ring-1" placeholder="you@example.com" />
      </label>
    </form>
  </div>
</Example>

```html
<label class="block">
  <span class="**after:content-['*']** **after:ml-0.5** **after:text-red-500** block text-sm font-medium text-slate-700">
    Email
  </span>
  <input type="email" name="email" class="mt-1 px-3 py-2 bg-white border shadow-sm border-slate-300 placeholder-slate-400 focus:outline-none focus:border-sky-500 focus:ring-sky-500 block w-full rounded-md sm:text-sm focus:ring-1" placeholder="you@example.com" />
</label>
```

Al usar estos modificadores, Tailwind agregará automáticamente `content: ''` de forma predeterminada para que no tengas que especificarlo a menos que desees un valor diferente:

<Example p="none">
  <div class="px-6">
    <div class="max-w-lg mx-auto bg-white shadow py-8 px-6">
      <blockquote class="text-2xl font-semibold italic text-center text-slate-900">
        Cuando te ves <span class="relative inline-block before:block before:absolute before:-inset-1 before:-skew-y-3 before:bg-pink-500"><span class="relative text-white">molesto</span></span> todo el tiempo, la gente piensa que estás ocupado.
      </blockquote>
    </div>
  </div>
</Example>

```html
<blockquote class="text-2xl font-semibold italic text-center text-slate-900">
  Cuando te ves
  <span class="**before:block** **before:absolute** **before:-inset-1** **before:-skew-y-3** **before:bg-pink-500** relative inline-block">
    <span class="relative text-white">molesto</span>
  </span>
  todo el tiempo, la gente piensa que estás ocupado.
</blockquote>
```

Vale la pena señalar que realmente no necesitas los pseudoelementos `::before` y `::after` para la mayoría de las cosas en los proyectos de Tailwind; por lo general, es más simple usar un elemento HTML real.

Por ejemplo, aquí está el mismo diseño de arriba pero usando un `<span>` en lugar del pseudo-elemento `::before`, que es un poco más fácil de leer y en realidad es menos código:

```html
  <blockquote class="text-2xl font-semibold italic text-center text-slate-900">
    Cuando te ves
    <span class="relative">
>     <span class="block absolute -inset-1 -skew-y-3 bg-pink-500" aria-hidden="true"></span>
      <span class="relative text-white">molesto</span>
    </span>
    todo el tiempo, la gente piensa que estás ocupado.
  </blockquote>
```

Guarda `before` y `after` para situaciones en las que es importante que el contenido del pseudoelemento no esté realmente en el DOM y el usuario no pueda seleccionarlo.

Ten en cuenta que si has deshabilitado nuestros [estilos base de verificación previa (preflight)](/docs/preflight), la propiedad de contenido no se establecerá en una cadena vacía de forma predeterminada, y deberás incluir `content-['']` en todo momento que uses los modificadores `before` y `after`.

<TipInfo>Si has deshabilitado la verificación previa, asegúrate de configurar el contenido manualmente</TipInfo>

```html
<div class="**before:content-['']** before:block ...">
  <!-- ... -->
</div>
```

## Modificadores personalizados

### Usar variantes arbitrarias

Al igual que los [valores arbitrarios](/docs/adding-custom-styles#using-arbitrary-values) te permiten usar valores personalizados con tus clases de utilidad, las variantes arbitrarias te permiten escribir modificadores de selector personalizados directamente en tu HTML.

Las variantes arbitrarias son solo cadenas de formato que representan el selector, entre corchetes. Por ejemplo, este modificador arbitrario selecciona un elemento solo cuando es el tercer hijo:

<SnippetGroup>

```html HTML
<ul role="list">
  {#each items as item}
    <li class="**[&:nth-child(3)]:underline**">{item}</li>
  {/each}
</ul>
```

```css Generated CSS
/* https://media.giphy.com/media/uPnKU86sFa2fm/giphy.gif */
.\[\&\:nth-child\(3\)\]\:underline:nth-child(3) {
  text-decoration-style: underline
}
```

</SnippetGroup>

La cadena de formato es la misma que usaría con la [API del complemento `addVariant`](/docs/plugins#adding-variants), con el `&` que representa el selector que se estás modificando.

Las variantes arbitrarias se pueden apilar con modificadores incorporados o entre sí, al igual que el resto de los modificadores en Tailwind:

<SnippetGroup>

```html HTML
<ul role="list">
  {#each items as item}
    <li class="**lg:[&:nth-child(3)]:hover:underline**">{item}</li>
  {/each}
</ul>
```

```css Generated CSS
/* https://media.giphy.com/media/Sd3cd0SrUKZEyWmAlM/giphy.gif */
@media (min-width: 1024px) {
  .lg\:\[\&\:nth-child\(3\)\]\:hover\:underline:hover:nth-child(3) {
    text-decoration-line: underline;
  }
}
```

</SnippetGroup>

Si necesitas espacios en tu selector, puedes usar un guión bajo. Por ejemplo, este modificador arbitrario selecciona todos los elementos `p` dentro del elemento donde has agregado la clase:

<SnippetGroup>

```html HTML
<div class="**[&_p]:mt-4**">
  <p>Lorem ipsum...</p>
  <ul>
    <li>
      <p>Lorem ipsum...</p>
    </li>
    <!-- ... -->
  </ul>
</div>
```

```css Generated CSS
.\[\&_p\]\:mt-4 p {
    margin-top: 1rem;
}
```

</SnippetGroup>

También puedes usar reglas-arroba como `@media` o `@supports` en variantes arbitrarias:

<SnippetGroup>

```html HTML
<div class="flex **[@supports(display:grid)]:grid**">
  <!-- ... -->
</div>
```

```css Generated CSS
/* https://media.giphy.com/media/sEms56zTGDx96/giphy.gif */
@supports (display: grid) {
  .\[\@supports\(display\:grid\)\]\:grid {
    text-decoration-style: underline
  }
}
```

</SnippetGroup>

Con los modificadores personalizados de regla-arroba, el marcador de posición `&` no es necesario, al igual que cuando se anida con un preprocesador.

Incluso puedes combinar reglas-arroba y modificadores de selector regulares al incluir el modificador selector entre llaves después de la regla arroba:


<SnippetGroup>

```html HTML
<button type="button" class="**[@media(any-hover:hover){&:hover}]:opacity-100**">
  <!-- ... -->
</button>
```

```css Generated CSS
/* https://media.giphy.com/media/l0IypeKl9NJhPFMrK/giphy.gif */
@media (any-hover: hover) {
  .\[\@media\(any-hover\:hover\)\{\&\:hover\}\]\:opacity-100:hover {
    opacity: 1
  }
}
```

</SnippetGroup>


### Crear un complemento

Si te encuentras usando el mismo modificador arbitrario varias veces en tu proyecto, podría valer la pena extraerlo a un complemento usando la API `addVariant`:

```js tailwind.config.js
let plugin = require('tailwindcss/plugin')

module.exports = {
  // ...
  plugins: [
    plugin(function ({ addVariant }) {
      // Añade una variante `third`, ej. `third:pb-0`
      addVariant('third', '&:nth-child(3)')
    })
  ]
}
```

Obtén más información en la documentación sobre [agregar variantes de complementos](/docs/plugins#adding-variants).

## Temas avanzados

### Uso con tus propias clases

Todos los modificadores de Tailwind están disponibles para usar con tus propias clases personalizadas siempre que los hayas definido en una de las [capas](/docs/adding-custom-styles#using-css-and-layer) de Tailwind o los hayas agregado usando un [complemento](/docs/adding-custom-styles#writing-plugins):


```css main.css
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer utilities {
  .content-auto {
    content-visibility: auto;
  }
}
```

```html HTML
<div class="**lg:content-auto**">
  <!-- ... -->
</div>
```

### Ordenar modificadores apilados

Al apilar modificadores, se aplican de adentro hacia afuera, como llamadas a funciones anidadas:

```js
// Estos modificadores:
'dark:group-hover:focus:opacity-100'

// ...se aplican así:
dark(groupHover(focus('opacity-100')))
```

En su mayor parte, esto en realidad no importa, pero hay algunas situaciones en las que el orden que usas en realidad genera un CSS significativamente diferente.

Por ejemplo, si tienes `darkMode` configurado en `class`, la combinación de los modificadores `dark` y `group-hover` genera un resultado diferente dependiendo del orden que use:

```css
/* dark:group-hover:opacity-100 */
.dark .group:hover .dark\:group-hover\:opacity-100 {
  opacity: 1;
}

/* group-hover:dark:opacity-100 */
.group:hover .dark .group-hover\:dark\:opacity-100 {
  opacity: 1;
}
```

En el primer ejemplo, el elemento `dark` debe ser padre del elemento `group`, pero en el segundo ejemplo se invierte.

Otro lugar en el que esto es importante es cuando se usan modificadores como `prose-headings` que se incluyen con el complemento de tipografía oficial:

```css
/* prose-headings:hover:underline */
.prose-headings\:hover\:underline:hover :is(:where(h1, h2, h3, h4, th)) {
  text-decoration: underline;
}

/* hover:prose-headings:underline */
.hover\:prose-headings\:underline :is(:where(h1, h2, h3, h4, th)):hover {
  text-decoration: underline;
}
```

En el primer ejemplo, cada encabezado individual se subraya cuando pasa el mouse sobre el artículo en sí, mientras que en el segundo ejemplo, cada encabezado solo se subraya cuando pasa el mouse sobre ese encabezado.

### Referencia de pseudoclase

Esta es una lista completa de ejemplos de todos los modificadores de pseudoclases incluidos en Tailwind para complementar la [documentación de pseudoclases](/docs/hover-focus-and-other-states#pseudo-clases) al comienzo de esta guía .

#### hover <small>(:hover)</small>

Aplica estilo a un elemento cuando el usuario se desplaza sobre él con el cursor del mouse usando el modificador `hover`:

```html
<div class="bg-black **hover:bg-white** ...">
  <!-- ... -->
</div>
```

#### focus <small>(:focus)</small>

Dale estilo a un elemento cuando tiene el foco con el modificador `focus`:

```html
<input class="border-gray-300 **focus:border-blue-400** ..." />
```

#### focus-within <small>(:focus-within)</small>

Dale estilo a un elemento cuando él o uno de sus descendientes tiene el foco usando el modificador `focus-within`:

```html
<div class="**focus-within:shadow-lg** ...">
  <input type="text" />
</div>
```

#### focus-visible <small>(:focus-visible)</small>

Dale estilo a un elemento cuando se haya enfocado usando el teclado usando el modificador `focus-visible`:

```html
<button class="focus:outline-none **focus-visible:ring** ...">
  Submit
</button>
```

#### active <small>(:active)</small>

Dale estilo a un elemento cuando se presiona usando el modificador `active`:

```html
<button class="bg-blue-500 **active:bg-blue-600** ...">
  Submit
</button>
```

#### visited <small>(:visited)</small>

Dale estilo a un enlace cuando ya ha sido visitado usando el modificador `visited`:

```html
<a href="https://seinfeldquotes.com" class="text-blue-600 **visited:text-purple-600** ...">
  Inspiration
</a>
```

#### target <small>(:target)</small>

Dale estilo a un elemento si su ID coincide con el fragmento de URL actual usando el modificador `target`:

```html
<div id="about" class="**target:shadow-lg** ...">
  <!-- ... -->
</div>
```

#### first <small>(:first-child)</small>

Dale estilo a un elemento si es el primer hijo usando el modificador `first`:

```html
<ul>
  {#each people as person}
    <li class="py-4 **first:pt-0** ...">
      <!-- ... -->
    </li>
  {/each}
</ul>
```

#### last <small>(:last-child)</small>

Dale estilo a un elemento si es el último hijo usando el modificador `last`:

```html
<ul>
  {#each people as person}
    <li class="py-4 **last:pb-0** ...">
      <!-- ... -->
    </li>
  {/each}
</ul>
```

#### only <small>(:only-child)</small>

Dale estilo a un elemento si es el único hijo usando el modificador `only`:

```html
<ul>
  {#each people as person}
    <li class="py-4 **only:py-0** ...">
      <!-- ... -->
    </li>
  {/each}
</ul>
```

#### odd <small>(:nth-child(odd))</small>

Dale estilo a un elemento si es un hijo impar usando el modificador `odd`:

```html
<table>
  {#each people as person}
    <tr class="bg-white **odd:bg-gray-100** ...">
      <!-- ... -->
    </tr>
  {/each}
</table>
```

#### even <small>(:nth-child(even))</small>

Dale estilo a un elemento si es un elemento par usando el modificador `even`:

```html
<table>
  {#each people as person}
    <tr class="bg-white **even:bg-gray-100** ...">
      <!-- ... -->
    </tr>
  {/each}
</table>
```

#### first-of-type <small>(:first-of-type)</small>

Dale estilo a un elemento si es el primer hijo de su tipo usando el modificador `first-of-type`:

```html
<nav>
  <img src="/logo.svg" alt="Vandelay Industries" />
  {#each links as link}
    <a href="#" class="ml-2 **first-of-type:ml-6** ...">
      <!-- ... -->
    </a>
  {/each}
</table>
```

#### last-of-type <small>(:last-of-type)</small>

Dale estilo a un elemento si es el último hijo de su tipo usando el modificador `last-of-type`:

```html
<nav>
  <img src="/logo.svg" alt="Vandelay Industries" />
  {#each links as link}
    <a href="#" class="mr-2 **last-of-type:mr-6** ...">
      <!-- ... -->
    </a>
  {/each}
  <button>More</button>
</table>
```

#### only-of-type <small>(:only-of-type)</small>

Dale estilo a un elemento si es el único hijo de su tipo usando el modificador `only-of-type`:

```html
<nav>
  <img src="/logo.svg" alt="Vandelay Industries" />
  {#each links as link}
    <a href="#" class="mx-2 **only-of-type:mx-6** ...">
      <!-- ... -->
    </a>
  {/each}
  <button>Más</button>
</table>
```

#### empty <small>(:empty)</small>

Dale estilo a un elemento si no tiene contenido usando el modificador `empty`:

```html
<ul>
  {#each people as person}
    <li class="**empty:hidden** ...">{person.hobby}</li>
  {/each}
</ul>
```

#### disabled <small>(:disabled)</small>

Dale estilo a una entrada cuando está deshabilitada usando el modificador `disabled`:

```html
<input class="**disabled:opacity-75** ..." />
```

#### enabled <small>(:enabled)</small>

Aplica estilo a una entrada cuando está habilitada usando el modificador `enabled`, más útil cuando solo deseas aplicar otro estilo cuando un elemento no está deshabilitado:

```html
<input class="**enabled:hover:border-gray-400** disabled:opacity-75 ..." />
```

#### checked <small>(:checked)</small>

Dale estilo a una casilla de verificación o botón de radio cuando esté marcado usando el modificador `checked`:

```html
<input type="checkbox" class="appearance-none **checked:bg-blue-500** ..." />
```

#### indeterminate <small>(:indeterminate)</small>

Dale estilo a una casilla de verificación o botón de radio en un estado indeterminado usando el modificador `indeterminate`:

```html
<input type="checkbox" class="appearance-none **indeterminate:bg-gray-300** ..." />
```

#### default <small>(:default)</small>

Aplica estilo a una opción, casilla de verificación o botón de radio que tenía el valor predeterminado cuando la página se cargó inicialmente usando el modificador `default`:

```html
<input type="checkbox" class="**default:ring-2** ..." />
```

#### required <small>(:required)</small>

Dale estilo a una entrada cuando sea necesario usando el modificador `required`:

```html
<input class="**required:border-red-500** ..." />
```

#### valid <small>(:valid)</small>

Dale estilo a una entrada cuando sea válida usando el modificador `valid`:

```html
<input class="**valid:border-green-500** ..." />
```

#### invalid <small>(:invalid)</small>

Aplica estilo a una entrada cuando no es válida usando el modificador `invalid`:

```html
<input class="**invalid:border-red-500** ..." />
```

#### in-range <small>(:in-range)</small>

Dale estilo a una entrada cuando su valor esté dentro de un límite de rango específico usando el modificador `in-range`:

```html
<input min="1" max="5" class="**in-range:border-green-500** ..." />
```

#### out-of-range <small>(:out-of-range)</small>

Dale estilo a una entrada cuando su valor está fuera de un límite de rango especificado usando el modificador `out-of-range`:

```html
<input min="1" max="5" class="**out-of-range:border-red-500** ..." />
```

#### placeholder-shown <small>(:placeholder-shown)</small>

Aplica estilo a una entrada cuando se muestra el placeholder usando el modificador `placeholder-shown`:

```html
<input class="**placeholder-shown:border-gray-500** ..." placeholder="you@example.com" />
```

#### autofill <small>(:autofill)</small>

Dale estilo a una entrada cuando el navegador la haya autocompletado usando el modificador `autofill`:

```html
<input class="**autofill:bg-yellow-200** ..." />
```

#### read-only <small>(:read-only)</small>

Dale estilo a una entrada cuando es de solo lectura usando el modificador `read-only`:

```html
<input class="**read-only:bg-gray-100** ..." />
```
