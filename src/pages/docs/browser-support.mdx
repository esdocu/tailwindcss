title: Compatibilidad con navegador
description: Comprender qué navegadores admite Tailwind y cómo administrar los prefijos de proveedores.
---

En general, Tailwind CSS v3.0 está diseñado y probado en las últimas versiones estables de Chrome, Firefox, Edge y Safari. No es compatible con ninguna versión de IE, incluido IE 11.

Si bien la mayoría de las funciones de Tailwind CSS funcionarán en todos los navegadores modernos, Tailwind también incluye API para varias funciones de última generación que aún no son compatibles con todos los navegadores, por ejemplo, la pseudoclase `:focus-visible` y las utilidades `backdrop-filter`.

Dado que Tailwind es un framework de tan bajo nivel, es fácil evitar estas funciones si no puedes usarlas simplemente al no usar la utilidad o el modificador que no es compatible, al igual que no usaría esas funciones de CSS en tu CSS .

La base de datos [Can I Use](https://caniuse.com/?search=focus-visible) es un excelente recurso cuando no estás seguro acerca de la compatibilidad con una característica CSS desconocida.

## Prefijos de proveedores

Muchas propiedades CSS requieren que los navegadores entiendan los prefijos de los proveedores, por ejemplo, `background-clip: text` necesita el prefijo `-webkit` para funcionar en la mayoría de los navegadores:

```css
.bg-clip-text {
  background-clip: text;
  -webkit-background-clip: text;
}
```

Si estás utilizando la herramienta Tailwind CLI, los prefijos de proveedores como este se agregarán automáticamente.

De lo contrario, le recomendamos que utiliza [Autoprefixer](https://github.com/postcss/autoprefixer), que es un complemento de PostCSS que agrega automáticamente cualquier prefijo de proveedor necesario en función de los navegadores que le indicas que necesita admitir.

Para usarlo, instálalo a través de npm:

```shell
npm install -D autoprefixer
```

Luego agrégalo al final de tu lista de complementos en tu configuración de PostCSS:

```js
module.exports = {
  plugins: [
    require('tailwindcss'),
    require('autoprefixer'),
  ]
}
```

Para obtener más información sobre cómo especificar los navegadores que necesita admitir, consulta [Browserslist](https://github.com/browserslist/browserslist), que es la forma estándar de definir los navegadores de destino en las herramientas de front-end.
